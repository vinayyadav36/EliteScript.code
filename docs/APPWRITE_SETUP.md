# Appwrite Integration Guide

SALTEDHASH utilizes Appwrite as its primary backend, particularly for data persistence (Products, Services, and Contact Leads).

## 1. Project Setup
- Create an Appwrite project at `cloud.appwrite.io`.
- Generate an **API Key** in the Appwrite console and add it to your `.env` as `APPWRITE_API_KEY` (Needed for the initialization script).

## 2. Infrastructure Deployment
We use a Node.js script to automatically initialize the entire Appwrite environment securely.

To configure collections, attributes, and seed initial data, run:
```sh
npm run init-appwrite
```

This will automatically create:
- **Database:** `saltedhash_public`
- **Collections:**
  - `products`: Product catalog data. Read permission `any`.
  - `services`: Studio services data. Read permission `any`.
  - `contact_leads`: Contact form submissions. Create permission `any`.

## 3. Storage Bucket
You must manually create the Storage bucket if it is not generated by the script:
1. Navigate to **Storage** in the Appwrite Console.
2. Create a bucket named **Product Assets** (ID: `product_assets`).
3. Set **Permissions**: `read("any")`.
4. Optionally, upload images and assign the File IDs to your Product documents in the Database.

## 4. Local Development Fallbacks
The application is resilient. If Appwrite is unavailable (e.g. invalid `PROJECT_ID` in `.env`), the application will:
- Warn in the developer console.
- Serve localized fallback content for the Products and Services pages.
- Handle Contact forms with a graceful offline warning state pointing users toward direct email or WhatsApp.
